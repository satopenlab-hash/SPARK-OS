# Innovation Skill

## Purpose

Use this skill to explore new ideas without confusing experiments with validated policy or production systems.

## Innovation rule

**Explore boldly. Label honestly.**

## Idea maturity labels

```text
Spark → Sketch → Experiment → Pilot → Validated → Reference
```

A Spark is not a promise.  
A Sketch is not an implementation.  
An Experiment is not evidence of impact.  
A Pilot is not a public system.

## Innovation capture template

```text
Idea:
Why it matters:
Who benefits:
What could go wrong:
Smallest safe experiment:
Evidence needed:
Human owner:
Status:
```

## Safe exploration boundaries

- Do not test with real student data by default.
- Do not imply legal, institutional, or scientific validation where none exists.
- Do not add speculative technology to classroom workflows without a teacher review path.
- Keep “Pandora Box” ideas in a separate clearly labeled experimental section.
- Convert only validated, governed ideas into core architecture.
