# Writing Skill

## Purpose

Use this skill for Markdown, README files, proposals, research briefs, technical documentation, policies, and public-facing explanations.

## Writing principles

- Be clear before being clever.
- Separate claims, evidence, interpretation, and proposal.
- Use short paragraphs and descriptive headings.
- Name uncertainty directly.
- Avoid inflated claims and unnecessary jargon.
- Use examples that are synthetic or privacy-safe.
- Link to supporting artifacts where possible.

## Standard artifact header

```markdown
# Title

**Status:** Genesis | Pilot | Validated | Reference | Certified  
**Owner:**  
**Last reviewed:**  
**Related layer:**  
**Human approval required:** Yes / No
```

## Public-facing OpenLab language

Prefer:
- “AI-assisted”
- “teacher-in-the-loop”
- “human-governed”
- “evidence-based”
- “privacy-respecting”
- “prototype”
- “recommendation”

Avoid:
- “fully autonomous assessment”
- “AI certification”
- “objective grading engine”
- “guaranteed learning outcomes”
- “blockchain credential economy”

## Quality checklist

Before finalizing:
- Is the purpose clear in the first paragraph?
- Are claims supportable?
- Is the audience obvious?
- Does it state limits and approvals?
- Is it easy for someone new to the repository to follow?
